User defined data type definitions

Prototype 
PWR_AppState
typedef enum
{
    PWR_APP_STATE_NO_ACTIVITY,
    PWR_APP_STATE_ADV,
    PWR_APP_STATE_CONN,
} PWR_AppState;
Description
  • PWR_APP_STATE_NO_ACTIVITY: application has no activity, low-power mode set in this case is VLLS0/VLLS1/PSWITCH according to application settings.
  • PWR_APP_STATE_ADV: application is advertising, low-power mode set in this case is VLLS2/3 for MCU and deep sleep shutdown for Bluetooth LE Link Layer.
  • PWR_APP_STATE_CONN: application is connected, low-power mode set in this case is VLLS2/3 for MCU and deep sleep shutdown for Bluetooth LE Link Layer. If Bluetooth LE Link Layer FIFO is not empty between two CONNECT events, low power mode applied is LLS3 instead of VLLS3 to prevent losing FIFO content.
